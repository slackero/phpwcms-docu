Updaten von phpwcms

Was ist zu beachten

Sichern der vorhandenen Dateien

Wenn Sie eine phpwcms Installation updaten wollen gehen Sie wie folgt vor.

Sichern Sie zuerst alle Dateien an denen Sie Veränderungen vorgenommen haben.
In der Regel handelt es sich um folgende Dateien bzw Ordner:

include/inc_conf/conf.inc.php
include/inc_conf/conf.template_default.inc.php
include/inc_conf/conf.indexpage.inc.php (seit 1.1-RC4 28.08.2004) template/inc_default/startup.html
template/inc_css/frontend.css
template/inc_cntpart

am besten ist es jedoch alle Ordner zu sichern.
Nun entpacken Sie die neue Version von phpwcms.

Empfehlenswert ist es, die neue Version Ihres phpwcms Systems mit den im Vorfeld gespeicherten Daten, (also das Backup des alten phpwcms- Systems) zu ersetzen.

Wenn Sie jedoch eine ältere Version von phpwcms installiert hatten (z.B. RC2) editieren Sie die (include/inc_conf/)conf.inc.php von Hand.

include/inc_conf/conf.inc.php
include/inc_conf/conf.template_default.inc.php
include/inc_conf/conf.indexpage.inc.php (seit 1.1-RC4 28.08.2004) template/inc_default/startup.html
template/inc_css/frontend.css
template/inc_cntpart


Im Zuge ständiger Erweiterungen wurden folgende Dateien stetig verändert und weiterentwickelt. Vergleichen Sie somit bitte auch die anderen Dateien wie conf.template_default.inc.php, frontend.css, da dort auch neue Einträge hinzugekommen sind.


TIP:
Zum optimalen Vergleich 2er Dateien empfehlen wir Ihnen das Freeware-Tool Winmerge. Ein besonderes Feature dieses Tools besteht darin 2 Dateien miteinander zu verbinden um somit einen optimalen textlichen Abgleich zu verschaffen.

Hier gehts zum Download
Nun laden Sie die entpackte und editierte neue Version hoch und überschreiben Ihre vorhandene phpwcms Installation.
Jetzt sollten Sie ein Datenbank Backup machen. Wenn Sie dies noch nicht gemacht haben, oder nicht wissen wie man eine Sicherung der Datenbank erstellt, folgen Sie bitte dem Link.

Datenbank sichern

Falls Sie schon eine Sicherung der Datenbank gemacht haben lesen Sie einfach weiter.
Nun kommen Sie zum eigentlichen Updateprozess. Rufen sie dazu die upgrade.php auf. Diese finden Sie im Setup Ordner Ihrer phpwcms Installation. (http://www.ihre_domain.com/setup/updgrade.php oder http://www.ihre_domain.com/unterordner/setup/upgrade.php)

Wenn Sie die upgrade.php aufgerufen haben sehen Sie folgende Übersicht (Siehe Bild links)


Im oberen Linken Bereich sehen folgende Felder:

db host: Ihr Datenbank Host (meist localhost)

db name: Ihr Datenbankname

db user: Der Datenbankbenutzer

db prepend:

select sql file: Hier wählen Sie die SQL Datei aus mit der phpwcms aktualisiert werden soll.
Drücken Sie nun auf das Auswahlfeld select sql file.

Nun sehen Sie folgende auswahl:

phpwcms_1.1-RC4_31-05-2004.sql
phpwcms_1.1-RC4_25-08-2004.sql
phpwcms_1.1-RC4_22-04-2004.sql
phpwcms_update_older.sql
phpwcms_1.1-RC4_13-08-2004.sql
phpwcms_1.1-RC4_08-06-2004.sql


Wählen Sie aus dieser Liste das erste Datum nach dem Release das Sie installiert haben. In diesem Falle wäre das phpwcms_1.1-RC4_13-08-2004.sql

Falls Sie eine Version installiert haben, die älter ist als die Datumsangaben in den SQL Dateien, starten Sie bitte mit phpwcms_update_older.sql.
Jetzt sehen Sie in der Textbox welche Einträge in der Datenbank vorgenommen werden. Drücken Sie auf Start Upgrade um die Neuerungen zu übernehmen.









Nachdem die Datenbank-Aktualisierung durchgeführt wurde, sehen Sie hier eine Übersicht der Aktualisierungen die vorgenommen wurden.

Um weitere Aktualisierungen durchzuführen klicken Sie einfach auf Choose another SQL file. Dann gelangen Sie zum Startbildschirm der Upgradefunktion zurück. Von dort aus können Sie wie gerade beschrieben weitere Updates, falls nötig und vorhanden, einspielen.




Falls Sie diesen Fehler angezeigt bekommen, haben Sie entweder eine aktuelle Version in der keine Updates nötig sind oder ein falsches SQL File ausgewählt.

Bitte klicken Sie dann wieder auf Choose another SQL file um das nächste Update - falls vorhanden - auszuwählen.
