Replacement Tags für die Navigation

{BROWSE:NEXT:Text:0 oder 1}

Verwendung: Ermöglicht ein Vorwärts-Blättern innerhalb der Seitenstruktur
Schreibweise: {BROWSE:NEXT:Seite vorwärts:1}
Anzeige: Seite vorwärts
Alternative: Setzen des Schalters von 1 auf 0 deaktiviert den Link:

{BROWSE:NEXT:Seite vorwärts:0}
Seite vorwärts
 

{BROWSE:PREV:Text:0 oder 1}

Verwendung: Ermöglicht ein Rückwärts-Blättern innerhalb der Seitenstruktur
Schreibweise: {BROWSE:PREV:Seite zurück:1}
Anzeige: Seite zurück
Alternative: Setzen des Schalters von 1 auf 0 deaktiviert den Link:

{BROWSE:PREV:Seite zurück:0}
Seite zurück
 

{BROWSE:UP:Text}

Verwendung: Ermöglicht ein Aufwärts-Springen innerhalb der Seitenstruktur zur nächsten, übergeordneten Kategorie
Schreibweise: {BROWSE:UP:Menü-Ebene aufwärts}
Anzeige: Menü-Ebene aufwärts
Bemerkung:
 

{NAV_TABLE_COLUMN}

Verwendung: Vertikales Navigationsmenü mit Javascript Mouseover-Effekt.
Die Anzahl der Unterebenen ist vom Programm nicht begrenzt.

Dieses äußerst mächtige Menüelement bildet die Menüstruktur der kompletten Site ab.
Schreibweise: {NAV_TABLE_COLUMN}

Bemerkung: Wegen der geometrischen Anordnung, sollte dieses Menü innerhalb der verwendeten Vorlage im Feld links oder rechts untergebracht werden. 

Das Aussehen des Menüs läßt sich in weitem Maße anpassen. 
Dazu ist innerhalb der Datei: 
config/phpwcms/conf.template_default.inc.php 
der Bereich:
// navigation table defaults 
zu modifizieren.

Verbunden mit einem horizontalen Menüelement, welches nur die erste Menüebene abbildet und verschiedenen Vorlagen für die unterschiedlichen Hauptebenen, läßt sich hiermit eine äußerst komplexe und optisch ansprechende Navigation realisieren.
 

{NAV_LIST_TOP}

Verwendung: Horizontales Menü zur Abbildung der ersten Menüebene.

Die Formatierung ist festgelegt innerhalb der Stylesheet-Datei: frontend.css  mit der  Class: list_top .
Schreibweise: {NAV_LIST_TOP}
Anzeige:
  • Home
  • Deutsch
  • English
Bemerkung: Soll die Form der Ausgabe den eigenen Vorstellungen angepaßt werden, bietet sich die folgenden Variante an!
 

{NAV_LIST_TOP:IndexName:ClassName}

Verwendung: Horizontales Menü zur Abbildung der ersten Menüebene.

Die Formatierung ist festgelegt innerhalb der Stylesheet-Datei: frontend.css  mit der  Class: list_top .

Die Bezeichnung der Index-Seite kann umbenannt werden.

Nach Eingabe einer eigenen Class ist die Formatierung der Anzeige den eigenen Wünschen weitestgehend anpaßbar. Diese Class muß natürlich in der frontend.css angelegt sein!
Schreibweise: {NAV_LIST_TOP:HOMEPAGE:}
Anzeige:
  • HOMEPAGE
  • Deutsch
  • English
Bemerkung: In diesem Beispiel wurde die Indexseite mit HOMEPAGE benannt. Es wurde keine eigene Class festgelegt.
 
Alternative: {NAV_LIST_TOP::}

  • Home
  • Deutsch
  • English

Hier wurde zusätzlich die Anzeige der Indexseite durch Weglassen eines Eintrags komplett unterdrückt.
 

{NAV_ROW}

Verwendung: Horizontales Menü zur Abbildung der ersten Menüebene, beginnend bei der Index-Seite.
Schreibweise: {NAV_ROW}
Anzeige: Home | Deutsch | English
Bemerkung: Für den Header- oder Footer-Bereich besonders geeignet.

Das Aussehen des Menüs läßt sich anpassen. 
Dazu ist innerhalb der Datei: 
config/phpwcms/conf.template_default.inc.php 
der Bereich:
// row based navigation
zu modifizieren.
 

{NAV_ROW:CURRENT:0 oder 1}

Verwendung: Horizontales Menü zur Abbildung der untergeordneten Menüebene, beginnend bei der aktuellen Seite.
Schreibweise: {NAV_ROW:CURRENT:1}
Anzeige: Navigations Tags | Test Level 1 | Test Level 2

Schalter ist ein: Die aktuelle Seite wird mit angezeigt!
Bemerkung: Für den Header- oder Footer-Bereich besonders geeignet.

Das Aussehen des Menüs läßt sich anpassen. 
Dazu ist innerhalb der Datei: 
config/phpwcms/conf.template_default.inc.php 
der Bereich:
// row based navigation
zu modifizieren.
 
Variante: {NAV_ROW:CURRENT:0}

Test Level 1 | Test Level 2

Schalter ist aus: Die aktuelle Seite wird nicht mit angezeigt!
 

{NAV_ROW:INT:0 oder 1}

Verwendung: Horizontales Menü zur Abbildung der untergeordneten Menüebene, beginnend unterhalb der angegebnen Kategorie-ID. Desweiten kann man mit 0|1 bestimmen ob der Aktuelle Seitenname mit Angezeigt wird.
Schreibweise: {NAV_ROW:INT:1}
Anzeige:

Navigations Tags | Test Level 1 | Test Level 2

Schalter ist ein: Die aktuelle Seite wird mit angezeigt!

 
Variante: {NAV_ROW:92:0}

Test Level 1 | Test Level 2

Schalter ist aus: Die aktuelle Seite wird nicht mit angezeigt!
Bemerkung: Für den Header- oder Footer-Bereich besonders geeignet.

Das Aussehen des Menüs läßt sich anpassen. 
Dazu ist innerhalb der Datei: 
config/phpwcms/conf.template_default.inc.php 
der Bereich:
// row based navigation
zu modifizieren.
 
 

{NAV_LIST_CURRENT}

Verwendung: Vertikales Menüelement. Es können nur die Kategorien unmittelbar unterhalb der aktuellen Menüebene sowie die übergeordnete Kategorie angesteuert werden.

Die Formatierung ist festgelegt innerhalb der Stylesheet-Datei: frontend.css  mit der  Class: list_level
Schreibweise: {NAV_LIST_CURRENT}
Anzeige:
  • Navigations Tags
  • Test Level 1
  • Test Level 2
Bemerkung:
 

{NAV_LIST_CURRENT:0 oder 1:IndexName:list_class}

Verwendung: Vertikales Menüelement. Es können nur die Kategorien unmittelbar unterhalb der aktuellen Menüebene sowie die übergeordnete Kategorie angesteuert werden.

Die Anzeige der übergeordneten Seite kann unterdrückt werden.
Die Bezeichnung der übergeordneten Seite kann umbenannt werden.
Die Formatierung ist festgelegt innerhalb der Stylesheet-Datei: frontend.css  mit der  Class: list_level .
Nach Eingabe einer eigenen Class ist die Formatierung der Anzeige den eigenen Wünschen weitestgehend anpaßbar. Diese Class muß natürlich in der frontend.css angelegt sein!
Schreibweise: {NAV_LIST_CURRENT:1:übergeordnete Seite:}
Anzeige:
  • übergeordnete Seite
  • Test Level 1
  • Test Level 2
Bemerkung: Hier wurde die aktuelle Seite in übergeordnete Seite umbenannt.
Es wurde keine eigene Class festgelegt.
 
Alternative: {NAV_LIST_CURRENT:0:übergeordnete Seite:}

  • Test Level 1
  • Test Level 2

(Damit wird die aktuelle Seite unterdrückt)
