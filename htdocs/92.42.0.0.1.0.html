Replacement Tags für die Navigation

RT: {BROWSE:NEXT:Text:0 oder 1}
Verwendung: Ermöglicht ein Vorwärtsblättern innerhalb einer Seitenstrukturebene.

Dieser Tag erlaubt das Vorwärtsblättern oder 'paging' von separaten Artikeln innerhalb einer Seitenstruktur (Kategorie).
Mit anderen Worten, wenn sich mehrere Artikel in einer Ebene befinden, macht es dieser Tag möglich einen automatischen Link zum nächsten Artikel innerhalb dieser Kategorie zu generieren. (Ebene=Kategorie).
Ist nur ein Artikel je Ebene vorhanden (häufiger Fall) wird von Ebene zu Ebene verzweigt (siehe folgende Erklärung).

- Beim Blättern über den letzten Artikel in der Ebene hinaus wird zum Artikel in der nächstmöglichen Ebene verzweigt. (Also keine Schleife!).
Schreibweise: {BROWSE:NEXT:Text:0 oder 1}
Ausgabe-Format: Seite vorwärts
Bemerkung: Setzen des Schalters von 1 auf 0 deaktiviert den Link:

{BROWSE:NEXT:Seite vorwärts:0}

Seite vorwärts
RT: {BROWSE:PREV:Text:0 oder 1}
Verwendung: Ermöglicht ein Rückwärtsblättern innerhalb einer Seitenstrukturebene.

Analog zu Vorwärtsblättern (siehe oben).
Schreibweise: {BROWSE:PREV:Text:0 oder 1}
Ausgabe-Format: Seite zurück
Bemerkung:

Setzen des Schalters von 1 auf 0 deaktiviert den Link:

{BROWSE:PREV:Seite zurück:0}

Seite zurück
RT: {BROWSE:UP:Text}
Verwendung: Ermöglicht ein Aufwärts-Springen innerhalb der Seitenstruktur zum ersten Artikel der nächsten, übergeordneten Kategorie.
Schreibweise: {BROWSE:UP:Text}
Ausgabe-Format: Menü-Ebene aufwärts
Bemerkung: --
RT: {NAV_TABLE_COLUMN}
Verwendung: Vertikales Navigationsmenü mit Javascript Mouseover-Effekt. Die Anzahl der Unterebenen ist vom Programm nicht begrenzt. Dieses äußerst mächtige Menüelement bildet die Menüstruktur der kompletten Site ab.
Schreibweise: {NAV_TABLE_COLUMN}
Ausgabe-Format:

Deutsch

Bemerkung:

Wegen der geometrischen Anordnung, sollte dieses Menü innerhalb der verwendeten Vorlage im Feld links oder rechts untergebracht werden. 

Das Aussehen des Menüs läßt sich in weitem Maße anpassen. 
Dazu ist innerhalb der Datei config/phpwcms/conf.template_default.inc.php 
der Bereich // navigation table defaults zu modifizieren.

Verbunden mit einem horizontalen Menüelement, welches nur die erste Menüebene abbildet und verschiedenen Vorlagen für die unterschiedlichen Hauptebenen, läßt sich hiermit eine äußerst komplexe und optisch ansprechende Navigation realisieren.
RT: {NAV_LIST_TOP}
Verwendung: Horizontales Menü zur Abbildung der ersten Menüebene. Die Formatierung ist festgelegt innerhalb der Stylesheet-Datei: frontend.css mit der Class: list_top .
Schreibweise: {NAV_LIST_TOP}
Ausgabe-Format:
  • Home
  • Deutsch
  • English
Bemerkung:

Soll die Form der Ausgabe den eigenen Vorstellungen angepaßt werden, bietet sich die folgenden Variante an!
RT: {NAV_LIST_TOP:IndexName:ClassName}
Verwendung: Horizontales Menü zur Abbildung der ersten Menüebene. Die Formatierung ist festgelegt innerhalb der Stylesheet-Datei: frontend.css mit der Class: list_top . Die Bezeichnung der Index-Seite kann umbenannt werden. Nach Eingabe einer eigenen Class ist die Formatierung der Anzeige den eigenen Wünschen weitestgehend anpaßbar. Diese Class muß natürlich in der frontend.css angelegt sein!
Schreibweise: {NAV_LIST_TOP:IndexName:ClassName}
Ausgabe-Format:
  • HOMEPAGE
  • Deutsch
  • English
Bemerkung:

In diesem Beispiel wurde die Indexseite mit HOMEPAGE benannt. Es wurde keine eigene Class festgelegt.

Alternativ:

{NAV_LIST_TOP::}

  • Home
  • Deutsch
  • English

Hier wurde zusätzlich die Anzeige der Indexseite durch Weglassen eines Eintrags komplett unterdrückt.
RT: {NAV_ROW}
Verwendung: Horizontales Menü zur Abbildung der ersten Menüebene, beginnend bei der Index-Seite.
Schreibweise: {NAV_ROW}

RT: {NAV_LIST_CURRENT:0 oder 1:IndexName:list_class}
Verwendung: Vertikales Menüelement. Es können nur die Kategorien unmittelbar unterhalb der aktuellen Menüebene sowie die übergeordnete Kategorie angesteuert werden.

Die Anzeige der übergeordneten Seite kann unterdrückt werden. Die Bezeichnung der übergeordneten Seite kann umbenannt werden. Die Formatierung ist festgelegt innerhalb der Stylesheet-Datei: frontend.css mit der Class: list_level .

Nach Eingabe einer eigenen Class ist die Formatierung der Anzeige den eigenen Wünschen weitestgehend anpaßbar. Diese Class muß natürlich in der frontend.css angelegt sein!
Schreibweise: {NAV_LIST_CURRENT:1:übergeordnete Seite:}
Ausgabe-Format:
  • übergeordnete Seite
Alternativ:
{NAV_LIST_CURRENT:0:übergeordnete Seite:} In diesem Beispiel wurde der Schalter auf 0 gesetzt: (Damit wird die aktuelle Seite unterdrückt)
Bemerkung:

Hier wurde die aktuelle Seite in übergeordnete Seite umbenannt. Es wurde keine eigene Class festgelegt.
