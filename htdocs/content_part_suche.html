Content Part "Suche"

Content Part "Suche"

Ergebnisse:

- pro Seite (optional): Anzahl anzuzeigender Ergebnisse je Seite. Wenn das Feld leer ist, werden alle Ergebnisse auf einer Seite dargestellt. 

- Öffne neues Fenster (optional): Hier können Sie wählen, dass ein neues Fenster geöffnet wird, um die Ergebnisse anzuzeigen. Ansonsten werden die Ergebnisse auf der Suchseite dargestellt.

- max. Anzahl Worte (Summary): Anzal der Worte, die im Suchergebnis ausgegeben werden.
Leer = 35 Worte
Seit V1.4.3 r373:
0 = keine Ausgabe des Schlüsseltextes (Summary)


- Highlight: Hervorheben der Suchwörter im Ergebnis.

- Minimale Anzahl Zeichen der Sucheingabe: Mindestanzahl der Zeichen die eingegeben werden müssen, damit die Suche beginnen kann.

Vorlage (optional): Hier können sie HTML-Code eintragen und damit die Standardeinstellungen überschreiben, um z.B. das Suchfeld individuell zu gestalten. Verwenden Sie dabei die folgenden beiden Platzhalter, damit die Suche funktioniert: 

search_input_field für den Namen des Sucheingabefeldes.
###search_input_action:ARTICLE_ID### für den Namen der Formularaktion, wobei ARTICLE_ID die Kennung des Artikels ist.

Verwenden Sie als Übertragungsmethode method="post".
Geben Sie value="" im Sucheingabefeld ein, wenn Sie wünschen, dass der Suchbegriff beim Aufruf der Ergebnisseite wieder im Suchfeld erscheint.

Beispielsweise könnte eine Vorlage also so aussehen:

<form action="index.php?aid=7" method="post">
<input type="text" name="search_input_field" size="10" maxlength="250" value="" style="font: normal 11px/11px sans-serif; border: solid 1px #94A4BF; background: #727A89; padding: 2px; color: #FFCC00;">
<input type="submit" value="Go" style="font-size:10px;">
</form>

Eingabe (optional):
Hier können Sie bestimmen, was als Beschreibung vor dem Suchfeld steht. Wenn Sie dieses Feld leer lassen, wird nichts vor dem Suchfeld angegeben. 

Button (optional):
Hier können Sie bestimmen, was auf der Taste zum Auslösen der Suche steht. Wenn Sie dieses Feld leer lassen wird „Search“ verwendet. 

Ergebnis (optional):
Hier können Sie bestimmen, was als Ergebnisüberschrift ausgegeben werden soll. Wenn Sie dieses Feld leer lassen wird lediglich die Anzahl der Treffer als Überschrift ausgegeben, also z.B. 1-10 

CSS Klassel (optional):
Hier können Sie eine eigene Stilvorlage für die Gestaltung des Suchfeldes, der Suchtaste bzw. der Ergebnisüberschrift bestimmen. Stilvorlagen können nur vom Administrator verändert werden. Sie werden weiter unten ausführlicher behandelt. 

 Weitere/Vorige (optional):
Hier können Sie bestimmen, was als Text zum Vor- bzw. Zurückblättern durch mehrere Ergebnisseiten angezeigt werden soll.

Suchnavi (optional):
Hier können Sie bestimmen, wie und ob eine Navigation generiert wird wenn mehrere Suchergebnisse angezeigt werden.
- Mit den darunterliegenden Feldern ([immer zeigen] [oben] [unten] usw. können Sie die Position bzw. das Verhalten steuern.
- In das Eingabefeld können die Steuerparameter für die Suchnavigation eingetragen werden.
Bsp. {PREV:Zurück}, Seite #/##, Ergebnis ###-####, {NAVI:123}, {NAVI:1-3}, {NEXT:Vorwärts}

Ausrichtung:
Hier können Sie wählen, wie das Suchformular auf der Artikelseite ausgerichtet sein soll. 

Intro (optional):
Hier können Sie einen Text eingeben, der beim Aufrufen der Suchseite vor dem Suchformular angezeigt werden soll. 

Ergebnis (optional):
Hier können Sie einen Text eingeben, der auf der Ergebnisseite vor dem Suchfeld ausgegeben werden soll. 

Kein Ergebnis (optional):
Hier können Sie einen Text eingeben, der vor dem Suchfeld ausgegeben wird, wenn eine Suche zu keinem Ergebnis geführt hat.

Beispiel einer eigenen Suchvorlage:
Dieses Beispiel können Sie in der Seiten-Vorlage an passender Stelle platziert.

<form action="index.php?search" method="post" name="searchFE" onsubmit="s=document.getElementById('customSearchInput');if(s.value==' Search')return false;">
<table width="100%" border="0" cellpadding="0" cellspacing="0">
<tr>
<td>
<input type="text" name="search_input_field" size="20" maxlength="200" id="customSearchInput" class="customSearchInput" value="" onfocus="if(this.value=='Search')this.value='';" onblur="if(this.value=='')this.value='Search';"/>
</td>
<td>
<input type="image" name="search_go" onfocus="this.blur();" title="Search" src="/template/lang/all/submit.gif" />
</td>
</tr>
</table>
</form>

Wobei <form action="index.php?search" ...>

der URL entspricht, unter der die Suche tatsächlich zu finden ist. Also darauf achten! Sie  können auch den Artikel direkt anspringen. Im Bsp. wurde das Alias "search" für eine Seitenebene gewählt.
Artikel werden mit (neuere Versionen) <form action="index.php?aid=%ARTIKEL-ID%" ...> angesprochen.
