Neue Seitenstruktur erstellen / bearbeiten

Seitenstruktur optionen

 Nachdem Sie auf gedrückt haben, um eine neue Struktur zu erstellen, gelangen sie zu folgender Eingabemaske.
Feldbeschreibungen
Seitenebenen Überschrift    Bestimmen Sie hier den Titel der Ebene.
Alias der Seitenebene

Hier bestimmen Sie den Alias der Strukturebene, dieser wird zum direktem Ansprechen der Ebene benötigt.

Beispiel: www.ihre_domain.tld/index.php?alias

Wenn Sie das Rewrite verwenden, wird aus dem Alias auch der Seitentitel erzeugt.

Beispiel: www.ihre_domain.tld/alias.html
Seitentitel  Sie können noch separat einen anderen Seitentitel erzeugen. Dieser erscheint dann in der Titelleiste des Browsers. Was in der Titelleiste angezeigt wird, bestimmen Sie in Ihrem Template unter "Titel hinzufügen:"
Weiterleiten Hier besteht die Möglichkeit, in eine andere Struktur oder auch auf externen Inhalt weiterzuleiten.

Wenn Sie seitenintern weiterleiten wollen, empfiehlt sich der {SITE} Tag.

Wenn Sie auf eine externe Seite weiterleiten wollen, geben sie hier einfach die URL der externen Seite ein, gefolgt von einem _blank. 

Hinweis:
Verwenden Sie für interne Weiterleitungen immer folgende Schreibweise index.php?ihr_alias
Beschreibung der Seitenebene: Hier können Sie für andere Redakteure noch eine Beschreibung der Ebene eintragen. Dieser ist nicht für die Besucher der Seite sichtbar.
Vorlage: Wählen Sie hier aus, welches Template für die neue Struktur verwendet werden soll.
Einstellungen: Hier ist es möglich, eine andere Vorlage für das Gesamte Aussehen der Seite zu wählen. Dazu liegt unter "template/inc_settings/template_default/" die Datei "custom_settings.php" diese Datei enthält die gleichen Werte wie die conf.template_default.php, die Sie im Config Verzeichnis von phpwcms finden.

Hinweis:
Bitte beachten Sie, wenn Sie Änderungen an der "custom_settings.php" vornehmen, diese auch umzubenennen, da bei einem Update die Datei überschrieben werden könnte.
Anzahl Top Artikel Hier bestimmen Sie, wie viele Artikel mit Schlagtext auf der Seite angezeigt werden sollen. Der Standard ist "-1" und bedeutet, dass alle Artikel mit  Schlagtext in dieser Ebene angezeigt werden.

Wenn Sie jedoch einen Artikel paginieren wollen müssen Sie hier die Anzahl der Artikel eintragen oder höher. Wenn Sie "-1" dort stehen lassen funktioniert die Paginierung nicht.
Paginierung: Es ist möglich, einen oder mehrere Artikel auf  mehrere Seiten aufzuteilen.

Mit dem Wert, den Sie in das Feld "Artikel je Seite" eintragen bestimmen Sie, wieviele Artikel je Seite angezeigt werden.
Anordnung zugeordneter Artikel:

Hier können Sie bestimmen, in welcher Sortierung die Artikel mit Schlagtext aufgeführt werden.

Folgende Einstellungen sind möglich:

manuell (Pfeil auf/ab)
Erstellungsdatum
Änderungsdatum
Startdatum
Enddatum
Artikeltitel

Außerdem können Sie durch Auswählen von

aufsteigend
absteigend 

die Reihenfolge bestimmen.
Content Part Selektion: Hier haben Sie die Möglichkeit, als Administrator zu bestimmen welche Content Parts in dieser Ebene zur Verfügung stehen. Wählen Sie keinen aus, werden alle angezeigt.

Wählen Sie dazu den entsprechenden Content Part im linken Fenster aus und fügen Sie ihn mit hinzu. Entfernen können Sie diesen wieder mit .
Frontend Menüstatus Sie haben hier zwei Möglichkeiten: 

versteckt: Wenn Sie hier den Haken setzen, wird diese Struktur für Besucher nicht in der Navigation angezeigt

sichtbar, wenn aktiv:

sichtbar nur für angemeldete Benutzer: Hat im Moment noch keine Funktion
Cache Wenn Sie hier den Haken setzen, wird die Seite nicht vom Cache berücksichtigt. Ansonsten können Sie bestimmen, nach welcher Zeit die Seite neu in den Cache eingelesen wird. Es ist ein Zeitraum von 1min bis 1 Monat möglich.
Suche Hier können Sie bestimmen, ob der Artikelinhalt mit in die Suche einbezogen werden soll.
Standard ist, dass der Inhalt gefunden werden kann. Durch Setzen des Hakens wird der Inhalt bei der Suche ausgeschlossen.
sichtbar / öffentlich Um die Struktur für Besucher sichtbar zu machen, setzen Sie bitte bei sichtbar und öffentlich einen Haken.
SSL Um SSL nutzen zu können benötigen Sie ein SSL Zertifikat. Sollten Sie eines besitzen, ändern Sie bitte die nötigen Einträge in der (config/phpwcms/)conf.inc.php unter dem Punkt "// dynamic ssl encryption engine"
Sitemap Hier können Sie durch Setzen den Hakens bestimmen, ob der Artikel in der Sitemap angezeigt werden soll.
