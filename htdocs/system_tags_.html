PHP

RT: [PHP] mein_PHP_code ; mehr_PHP_code [/PHP]
Verwendung: Hiermit kann benötigter PHP-Code direkt eingegeben werden.
Schreibweise: [PHP] mein_PHP_code ; mehr_PHP_code [/PHP]
Ausgabe-Format: --
Bemerkung:

Eignet sich insbesondere für kurze PHP-Anweisungen.
RT: {PHP:mein_externe_PHP_Datei.php}
Verwendung: Hiermit wird ein komplettes PHP-Script eingebunden.
Schreibweise: {PHP:mein_externe_PHP_Datei.php}
Ausgabe-Format: --
Bemerkung:

Es ist der Dateiname einschließlich der notwendigen Pfadangabe einzugeben!
Längere PHP-Scripte sollten ggf. innerhalb eines iframe in phpWCMS eingebunden werden.

Nützliches ...

RT: [BACK] Text oder Bild [/BACK]
Verwendung: Erzeugt eine Zurück-Funktion. Bei Klick auf den Text oder das Icon springt die Anzeige auf die vorhergehende Seite der Historie.
Schreibweise: [BACK] Text oder Bild [/BACK]
Ausgabe-Format: eine Seite zurück
Bemerkung:

Anstelle des Textes kann auch eine Grafik eingefügt werden,
in der Form:

[BACK]{IMAGE:back_link_.gif}[/BACK]
RT: [BOOKMARK] Text oder Bild [/BOOKMARK]
Verwendung: Erstellt einen Link zum Öffnen des Fensters Zu Favoriten hinzufügen
Schreibweise: [BOOKMARK] Text oder Bild [/BOOKMARK]
Ausgabe-Format:  
Bemerkung:

Anstelle des Textes kann auch eine Grafik eingefügt werden in der Form:

[BOOKMARK]{IMAGE:bookmark_link_.gif}[/BOOKMARK]

 
RT: [PRINT_PDF]Text oder Bild[/PRINT_PDF]
Verwendung: Erstellt von der Aktuellen Seite eine PDF Datei. Diese Funktion ist verfügar ab Version 1.3.2
Schreibweise: [PRINT_PDF]Text oder Bild[/PRINT_PDF]
Ausgabe-Format:
Bemerkung:

Anstelle des Textes kann auch eine Grafik eingefügt werden, in der Form:

[PRINT_PDF]{IMAGE:icon_pdf.gif}[/PRINT_PDF]
 

Hinweis:
Die Ausgabe der PDF Datei können Sie über eine vorlagen/template Datei steuern.
Die Datei finden Sie im Ordner template/inc_default/pdf.tmpl
RT: [PRINT] Text oder Bild [/PRINT]
Verwendung: Erzeugt einen Link zu einer druckerfreundlichen Ansicht, in der nur die Inhalts-Elemente des Content-Bereichs aufgeführt sind. Alle Header-, Footer- und Navigationselemente sind entfernt.
Schreibweise: [PRINT] Text oder Bild [/PRINT]
Ausgabe-Format:
Bemerkung:

Anstelle des Textes kann auch eine Grafik eingefügt werden, in der Form:


[PRINT]{IMAGE:print_link_.gif}[/PRINT]

 

Tipp:

Um das Druckergebnis zu optimieren verwenden Sie in Ihrer Vorlage:

        <!--NO_PRINT_START//-->

        <!--NO_PRINT_END//-->

Der Bereich der sich dazwischen befindet wird beim Druck nicht berücksichtigt. Diese Tag's können mehrfach verwendet werden.
RT: [RSS all] Text oder Bild [/RSS]
Verwendung: Mit diesem Replacement Tag können Sie Ihren Inhalt als RSS Feed anbieten
Schreibweise: [RSS all] Text oder Bild [/RSS]
Ausgabe-Format: RSS
Bemerkung:

Folgende Einstellungen sind möglich:

[RSS all]RSS-Feed[/RSS]
gibt die komplette Webseite als RSS Feed aus.

[RSS INT]RSS-Feed[/RSS]
ersetzen Sie INT durch eine ID einer Strukturebene und es wird nur diese als RSS Feed ausgegeben.

[RSS ALIAS]RSS-Feed[/RSS]
ersetzen Sie ALIAS durch einen auf Ihrer Webseite bestehenden Alias und es wird nur diese Ebene als RSS Feed ausgegeben.

Anstelle des Textes kann auch eine Grafik eingefügt werden.

[RSS all]{IMAGE:rss_valid.gif}[/RSS]
RT: [download=ID,ID,ID]Datei-1 Datei-2 Datei-3[/download]
Verwendung: Mit diesem Replacement Tag können Sie Dateien aus der Dateizentrale direkt zum Download anbieten. (Verfügbar ab V1.39 r187 - 21.05.2008)
Schreibweise: [download=ID,ID,ID]Datei-1 Datei-2 Datei-3[/download]
Ausgabe-Format:
Bemerkung: Die IDs werden in der Dateizentrale ermittelt. Beim Überfahren der Datei/des Bildes mit der Maus wird die ID angezeigt.

Inline-Download wird eingeschaltet. Die Ausgabe wird durch das Template /template/inc_default/filelist.html gesteuert.


Folgende Einstellungen sind möglich:

[download=ID,ID,ID]Datei-1 Datei-2 Datei-3[/download]
Bietet die drei Dateien Datei-1 Datei-2 Datei-3 zum Download an.

[download=ID,ID,ID/]
Kurze Schreibweise ohne Namen.

Vorlagen

RT: {BREADCRUMB}
Verwendung: Erzeugt ein sog. Brotkrümel-Menü, welches dem Besucher zeigt, an welcher Stelle der Navigation er sich befindet. Ermöglicht ein gezieltes, schrittweises Zurückgehen.
Schreibweise: {BREADCRUMB} oder {BREADCRUMB:ID}
Ausgabe-Format: Home > Deutsch > Die Docu > Admin > Replacement-Tags > System Tags oder (ID=309) Die Docu > Admin > Replacement-Tags > System Tags
Bemerkung:

Sinnvoll bei Sites mit einer tiefen Navigations-Struktur.
Wird normalerweise in der Vorlage eingesetzt.

Bei Verwendung der ID muss der aktive Link immer auf oder hinter der ID in der Seitenstruktur liegen, sonst wird ein Fallback generiert mit der Ausgabe von {BREADCRUMB}
RT: {CONTENT}
Verwendung: Verantwortlich für die Wiedegabe aller Inhalts-Elemente. Kann nur einmal vergeben werden! Steht sinnvoller weise innerhalb der Vorlage im Feld Haupt.
Schreibweise: {CONTENT}
Ausgabe-Format: --
Bemerkung:

Sollte keine Vorlage eingegeben werden, ist diese RT im Haupt-Feld vorgegeben!

Neben diesem RT, gibt es weitere "Vorlagen-RT's",  die ebenfalls in den Vorlagen zum Einsatz kommen sollten:

  • {HEADER}
  • {RIGHT}
  • {LEFT}
  • {FOOTER}

Auch hier muss bemerkt werden, dass sie nur einmalig verwendet werden können.

URL

RT: {SITE}
Verwendung: Gibt die Aktuelle URL aus.
Schreibweise: {SITE}index.php?hallo
Ausgabe-Format: ./
Bemerkung:

Kann vor allem bei Weiterleitungen benutzt werden. Sehr nützlich wenn man z.B. die Domain wechselt oder die Seite lokal entwirft.
RT: {URL:url_der_externen_site}
Verwendung: Hierdurch wird eine externe Internetseite eingebunden.
Schreibweise: {URL:url_der_externen_site}
Ausgabe-Format: --
Bemerkung:

Einschränkung: 
Nur der Bereich zwischen <BODY> und </BODY> wird übernommen!
RT: [PHP]echo FE_CURRENT_URL[/PHP]
Verwendung: Hierdurch wird die aktuelle URL im Content angezeigt.
Schreibweise: [PHP]echo FE_CURRENT_URL[/PHP] oder [PHP]echo abs_url()[/PHP]
Ausgabe-Format: deutsche_version.html?system_tags_
Bemerkung:

Einschränkung: 
Nur der Bereich zwischen <BODY> und </BODY> wird übernommen!
