Update der Version 1.2.x

Bevor Sie mit dem eigentlichen Update beginnen sichern Sie ALLE Dateien und machen Sie ein Datenbank Backup.

Wie sichert man eine Datenbank?

Datenbank sichern

1. Vorbereitungen

Nachdem Sie nun alles gesichert und die aktuelle Entwicklerversion (DEV) heruntergeladen und entpackt haben, editieren Sie bitte zuerst die conf.inc.php. Die conf.inc.php finden Sie ab sofort in folgendem Ordner: config/phpwcms/.
Bitte bearbeiten Sie die conf.inc.php, so dass die Einstellungen mit denen der alten Version übereinstimmen.

Darüberhinaus müssen folgende Dateien angepasst werden.

1. /config/phpwcms/conf.template_default.inc.php
2. /template/inc_css/frontend.css

Dies sind die Dateien, die sehr oft verändert werden. Sollten Sie an anderer Stelle noch Änderungen vorgenommen haben, passen Sie diese bitte auch an.

2. Dateien hochladen

Nun laden Sie die angepasste Entwicklerversion mit einem FTP Programm hoch und überschreiben dabei die existierende Version.

3. Datei- und Verzeichnis-Rechte überprüfen

Setzen Sie nun z.B. mit dem FTP Programm die Dateirechte, damit alle Funktionen von phpwcms auch gewährleistet sind.

Verzeichnis   phpwcms_ftp   777
Verzeichnis   phpwcms_filestorage   777
Verzeichnis   content   777
Verzeichnis   content/images   777
Verzeichnis   content/gt   777
Verzeichnis   content/form   777
Verzeichnis   content/rss   777
Verzeichnis   content/tmp   777
Verzeichnis   content/pages   777
         
Datei   template/inc_css/frontend.css   666
Datei   config/phpwcms/conf.indexpage.inc.php   666
Datei   template/inc_default/startup.php   666

4. Datenbank updaten

Nun kommen Sie zum eigentlichen Updateprozess. Rufen Sie dazu die upgrade.php auf. Diese finden Sie im Setup Ordner Ihrer phpwcms Installation. (http://www.ihre_domain.com/setup/upgrade.php oder http://www.ihre_domain.com/unterordner/setup/upgrade.php)

Wenn Sie die upgrade.php aufgerufen haben, sehen Sie folgende Übersicht (Siehe Bild links):Im oberen linken Bereich sehen Sie folgende Felder:

db host: Ihr Datenbank Host (meist localhost)

db name: Ihr Datenbankname

db user: Der Datenbankbenutzer

db prepend:

select sql file: Hier wählen Sie die SQL Datei aus, mit der phpwcms aktualisiert werden soll.

5. SQL-Update ausführen

Drücken Sie nun auf das Auswahlfeld select sql file.

Nun sehen Sie folgende Auswahl:

00__phpwcms_update_older.sql
01__phpwcms_1.1-RC4_22-04-2004.sql
02__phpwcms_1.1-RC4_31-05-2004.sql
03__phpwcms_1.1-RC4_08-06-2004.sql
04__phpwcms_1.1-RC4_13-08-004.sql
05__phpwcms_1.1-RC4_25-08-004.sql
06__phpwcms_1.1-RC4_15-10-004.sql
07__1.1.5-DEV_to_1.1.9-DEV.sql
07__27-08-2004_to_1.1.9-DEV.sql
08__1.1.9-DEV_to_1.2-DEV.sql 
09__1.2.1-DEV_to_1.2.2-DEV.sql
10__1.2.2-DEV_to_1.2.3-DEV.sql
11__1.2.3-DEV_to_1.2.4-DEV.sql
12__1.2.4-DEV_to_1.2.5-DEV.sql

Wählen Sie aus dieser Liste das erste Datum nach dem Release, das Sie installiert haben.


Falls Sie eine Version installiert haben, die älter ist als die Datumsangaben in den SQL Dateien, starten Sie bitte mit phpwcms_update_older.sql.

Jetzt sehen Sie in der Textbox, welche Einträge in der Datenbank vorgenommen werden. Drücken Sie auf Start Upgrade, um die Neuerungen zu übernehmen.

Nachdem die Datenbank-Aktualisierung durchgeführt wurde, sehen Sie hier eine Übersicht der Aktualisierungen, die vorgenommen wurden.

Um weitere Aktualisierungen durchzuführen, klicken Sie einfach auf Choose another SQL file. Dann gelangen Sie zum Startbildschirm der Upgradefunktion zurück. Von dort aus können Sie wie gerade beschrieben weitere Updates, falls nötig und vorhanden, einspielen.

Falls Sie diesen Fehler angezeigt bekommen, haben Sie entweder eine aktuelle Version, in der keine Updates nötig sind oder ein falsches SQL File ausgewählt.

Bitte klicken Sie dann wieder auf Choose another SQL file, um das nächste Update - falls vorhanden - auszuwählen.

WICHTIG!

Bitte vergessen Sie zum Abschluß nicht, das Setup Verzeichnis zu löschen oder umzubenennen, da ansonsten ein Sicherheitsrisiko besteht!
